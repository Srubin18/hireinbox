## PART 6: SPECIFIC HIREINBOX ISSUES

### Homepage Problems

1. **"Less noise. Better hires."** - What does this mean? No one knows.
2. **No product screenshot** - What am I buying?
3. **No social proof** - Why should I trust you?
4. **Three generic boxes** - Could be any SaaS template
5. **No pricing indication** - Will this cost R100 or R10,000?
6. **No demo video** - Show, don't tell

### Dashboard Problems

1. **Empty state is sad** - Just shows "no data"
2. **IMAP setup is confusing** - Users don't know what IMAP is
3. **No onboarding** - Users land lost
4. **Hidden role creation** - Key action is not prominent
5. **Pricing not shown** - Users don't know if they can afford this

### Candidate Flow Problems

1. **"Student vs Professional"** - Wrong dichotomy
2. **Video upsell goes nowhere** - Links to non-existent page
3. **No account created** - Can't return to results
4. **No follow-up** - One and done

### Talent Pool Problems

1. **Zero candidates** - The page is a lie
2. **Data not saved** - Forms submit to nowhere
3. **Fake browse page** - Shows hardcoded samples

## CONCLUSION

**HireInbox is not ready to compete.**

| Area | Minimum Viable | HireInbox Status |
|------|----------------|------------------|
| Trust | Some proof | None |
| Features | Core workflow complete | Screening only |
| Design | Professional, distinctive | Generic, AI-looking |
| UX | Clear, guided | Confusing, dead ends |
| Content | Real, specific | Vague, placeholder-feeling |

**The path forward:**

1. **Delete fake features** (done for recruiter portal)
2. **Perfect one workflow:** Employer creates role → receives CVs → AI screens → shortlists
3. **Get real users:** Even 5 paying customers changes everything
4. **Build trust signals:** Each customer = logo + testimonial + case study
5. **Invest in design:** Either hire a designer or use a premium template

**You have a working AI engine. Now you need a trustworthy wrapper around it.**
